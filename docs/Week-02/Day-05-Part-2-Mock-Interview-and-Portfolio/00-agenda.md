## What LLM engineering roles actually look for

Most job postings list 20 skills. In interviews, they test 5:

1. **Can you make an LLM do a specific thing reliably?** (prompt engineering, structured output, tool use)
2. **Can you build a production API around it?** (FastAPI, async, error handling, deployment)
3. **Can you make it fast and cheap?** (caching, batching, model selection, token budgets)
4. **Can you evaluate whether it works?** (metrics, RAGAS, human eval, A/B testing)
5. **Can you explain tradeoffs?** (RAG vs fine-tuning, LangGraph vs simple chains, streaming vs batch)

Every mock interview question maps to one of these five areas.
